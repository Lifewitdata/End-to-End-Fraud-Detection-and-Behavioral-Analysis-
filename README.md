# 💳 Payment Fraud Detection & Analytics: A Deep Dive into Mobile Money Transactions

## 🚀 Project Overview

Welcome to this end-to-end data science project focused on tackling the critical issue of payment fraud in mobile money transactions. With global financial losses from fraud exceeding **$32 billion annually**, building robust detection systems is not just important – it's essential. This project develops a complete fraud detection pipeline, from initial data profiling to machine learning model deployment and business impact quantification.

### 🤔 What Questions Did We Answer?

1.  **Fraud Patterns**: Which transaction types and behavioral patterns are most indicative of fraudulent activity?
2.  **Model Reliability**: Can we build a machine learning model that accurately detects fraud while keeping false alarms to a minimum?
3.  **Financial Impact**: What's the real-world financial benefit of our fraud detection capabilities?

## 📊 The Dataset: PaySim Mobile Money

We utilized the `PS_log.csv` dataset, a synthetic financial dataset generated by PaySim. It simulates mobile money transactions based on a sample of real-world aggregated data from a financial service. This dataset includes various transaction types (CASH_IN, CASH_OUT, DEBIT, PAYMENT, TRANSFER) and flags fraudulent transactions.

## 🛠️ Methodology & Key Steps

This project follows a structured approach, covering all major phases of a data science lifecycle:

1.  **Data Ingestion & SQL Profiling (with DuckDB)**: Employed DuckDB for efficient SQL-based data exploration and profiling directly on the CSV file, mimicking enterprise-grade big-data environments. This helped us understand schema, counts, and initial fraud rates.
2.  **Data Quality Validation & Cleansing**: Performed rigorous checks for missing values, duplicates, and business logic violations (like balance anomalies), ensuring a clean dataset for modeling.
